Semantic HTML & Layouts

Semantic HTML elements provide meaning to web content, making it easier for search engines and screen readers to understand.

1. What is Semantic HTML?

Semantic elements clearly define their purpose, unlike generic <div> and <span>.

2. Common Semantic Elements

  • <header> - Defines the page header.
  • <nav> - Contains navigation links.
  • <section> - Groups related content.
  • <article> - Represents self-contained content.
  • <aside> - Represents side content.
  • <footer> - Defines the page footer.

3. Example of a Semantic Layout

<header>
    <h1>My Website</h1>
</header>

<nav>
    <a href="#">Home</a>
    <a href="#">About</a>
</nav>

<main>
    <section>
        <article>
            <h2>Welcome!</h2>
            <p>This is an article inside a section.</p>
        </article>
    </section>
    <aside>
        <p>This is a sidebar.</p>
    </aside>
</main>

<footer>
    <p>Copyright Â© 2025</p>
</footer>

4. Benefits of Using Semantic HTML

  • Improves accessibility.
  • Helps search engines understand the content.
  • Makes code more readable and maintainable.

Using semantic HTML improves the structure and usability of web pages.
